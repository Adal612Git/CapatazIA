Experience Layer

Web ejecutiva para dueño y admin operativo para staff. Mobile app para experiencia premium. WhatsApp como canal principal del usuario VIP.

React + Expo + canal conversacional

Intelligence Layer

Mr. Shark interpreta intención patrimonial. Capataz AI convierte intención en tarea, workflow, recordatorio o aprobación.

LLM gateway + memoria + clasificación

Orchestration Layer

Motor de políticas, thresholds, approvals, escalamiento humano y handoff. Aquí vive el control fino del riesgo.

Policy engine + workflow runtime

Domain Services

Wallet, portfolio, approvals, operations, notifications, audit y canales desacoplados con contratos internos estables.

NestJS modular + eventos internos

Integration Layer

Adaptadores por proveedor para STP/SPEI, tastytrade, Apex futuro, WhatsApp y pasarelas. Cambio de proveedor sin reescribir negocio.

Adapters + provider abstraction

Data + Audit

Ledger, perfiles, eventos, decisiones, memoria de conversación, attachments y evidencia regulatoria. Nada crítico queda fuera del trail.

PostgreSQL + Redis + object storage
MVP Now

Cloudflare/Nginx + frontend React en Vercel o contenedor Nginx; API NestJS en contenedor; PostgreSQL administrado; Redis para sesiones, rate limits y colas ligeras; S3/MinIO para archivos y evidencia.

Entorno IA

Gateway propio para modelos, con proveedor configurable. Así Mr. Shark y Capataz no quedan amarrados a un solo LLM.

Webhook Hub

Entrada centralizada para WhatsApp, brokers y pagos. Firma, validación, idempotencia y auditoría primero; procesamiento después.

Observabilidad

Logs estructurados, métricas, trazas, alertas y tablero de eventos críticos. Cada comando sensible debe poder reconstruirse sin adivinanzas.

Escalamiento

Cuando el piloto crezca: separar workers, event bus, multi-env estricto y vault de secretos. El core de dominio no cambia; solo madura la infraestructura.

Narrativa regulatoria correcta

No venderlo como banco. No prometer rendimiento sobre saldo wallet. No presentar ejecución discrecional automática sin sponsor/licencia cerrada.

Control por umbrales

Transferencias, órdenes y aprobaciones sensibles con thresholds configurables, doble confirmación y opción de handoff humano.

Audit trail end-to-end

Quién pidió, quién aprobó, cuándo, desde qué canal, con qué contexto y contra qué adaptador externo se ejecutó.

Provider-agnostic real

Dominio desacoplado de proveedores. La interfaz conversa con Mr. Shark; los adaptadores hablan con tastytrade, Apex, STP o pasarelas.
